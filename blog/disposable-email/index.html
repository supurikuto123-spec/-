使い捨てメールとは？

使い捨てメール（捨てメアド／一時メール）は、短期間だけ受信できる仮のメールアドレスです。 サイト登録時の確認コード受信、アプリ検証、スパム回避などに使われます。多くは 登録不要・即時発行・Web上で受信確認が可能です。

仕組み

① 一時アドレスの発行

アクセス時にランダムなアドレス（例: xxxx@temp.example）を生成。エイリアス方式やキャッチオール方式が使われます。

② 受信と表示

そのアドレス宛のメールを一時ストレージに保存し、ブラウザからリアルタイム表示（数秒ごと自動更新）。一定時間・件数で自動削除。

種類と違い

方式概要向いている用途注意点
使い捨て（公開インボックス） 誰でも同じ受信箱を閲覧できる場合がある。登録不要で即利用。 検証・一回きりの登録確認 第三者も読める可能性。機密NG。
転送型（プライベート） 仮アドレス → 本アドレスへ転送。自分だけが受信。 ニュースレター試用、スパム分離 転送先の管理が必要。サービス側で禁止される場合あり。
エイリアス（通常メールの別名） Gmail等の「+タグ」や独自ドメインの別名で受信箱を振り分け。 長期運用、ラベリング管理 本アドレスのドメインが相手に見える。

主な用途

  • 会員登録・ダウンロード時の確認コード受信
  • 新サービスの試用や価格調査
  • スパム回避・メールマーケからの切り分け
  • 開発・テスト（サインアップ／通知動作の検証）

安全性とリスク

メリット

  • 本アドレスの露出を防ぎ、スパムを分離
  • 登録不要・即時で手早い
  • テスト環境の再現が容易

リスク

  • 公開インボックスの場合、第三者が閲覧可能
  • 運営側の保存期間・削除ポリシーに依存
  • パスワードリセット等に使うと乗っ取りリスク

重要・金銭・本人確認が絡むサービスでは使用しないのが鉄則。二段階認証や銀行・EC本会員、政府手続き、クラウドの管理者権限などはNGです。

使わない方がよい場面

  • 金融・フリマ・決済・各種KYC（本人確認）が必要なサービス
  • 仕事用の中核SaaS、クラウド管理者、ドメイン／証明書、課金主体のアカウント
  • 長期にわたり受信履歴が必要な場面、法的通知の受領

開発者向け：APIで自動化

tomy634.com は REST API を用意しています（共有APIキー配布中）。検証フローやツール連携に便利です。

例：cURLで新しい一時アドレスを取得

# 仮の例。API仕様に合わせてエンドポイントやパラメータを調整してください
curl -X POST https://tomy634.com/api/v1/address \
  -H "Authorization: Bearer <YOUR_API_KEY>" \
  -H "Content-Type: application/json" \
  -d '{ "ttl": 3600 }'
# → { "address": "abcd1234@tomy634.com", "expires_in": 3600 }

例：受信をポーリングする（Node.js）

import fetch from "node-fetch";

const API = "https://tomy634.com/api/v1";
const KEY = process.env.TOMY634_API_KEY;
const addr = "abcd1234@tomy634.com";

async function poll() {
  const res = await fetch(`${API}/inbox?address=${encodeURIComponent(addr)}`, {
    headers: { Authorization: `Bearer ${KEY}` }
  });
  const json = await res.json();
  for (const mail of json.messages) {
    console.log(mail.subject, mail.from, mail.receivedAt);
  }
}
setInterval(poll, 15000);

※APIキーの取り扱いには注意し、クライアント側へ直埋めしないでください。

FAQ

Q. どれくらい保存されますか？

サービスのポリシーによります。tomy634.com は最大件数・時間でローテーションし、古いメールは消えます。長期保存が必要な内容は通常メールをご利用ください。

Q. 同じアドレスを後日も使えますか？

原則として「一時的」です。恒常利用が必要なら転送型サービスやエイリアスをご検討ください。

Q. 一時メールは違法では？

多くの国で一時メール自体は違法ではありません。ただし各サイトの規約で禁止されている場合があり、悪用は当然NGです。

Q. パスワード再発行に使ってもいい？

おすすめしません。第三者に閲覧されるリスクや、後から受け取れない可能性があります。重要アカウントは本アドレス＋多要素認証で守りましょう。
